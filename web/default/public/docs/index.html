Actorus New API

Codex 与 Claude Code 接入文档

这里记录当前对外分发的 API 使用方式。用户只需要使用管理员发放的 new-api Token，不需要知道本机 CPA 或 Cloudflare Tunnel 的内部配置。

快速信息 获取 Token Codex 配置 Claude Code 配置 模型与价格 常见问题

快速信息

控制台

https://newapi.actorus.top

OpenAI-compatible Base URL

https://newapi.actorus.top/v1

Claude-compatible API Endpoint

https://newapi.actorus.top/v1/messages

鉴权方式

Authorization: Bearer sk-... 或 x-api-key: sk-...

对外只使用 https://newapi.actorus.top。不要使用 dev.actorus.top，那是旧的 CPA 直连入口，无法按用户统计额度。

获取 Token

  1. 联系管理员创建账号并分配额度。
  2. 登录 new-api 控制台。
  3. 进入“令牌管理”，创建一个新 Token。
  4. 复制形如 sk-... 的 Token，用于下面的 Codex 或 Claude Code 配置。

Token 与用户额度绑定。不要把自己的 Token 发给其他人，否则用量会计入你的账号。

Codex 配置

Codex 走 OpenAI-compatible 接口。Base URL 要包含 /v1。

Windows PowerShell 临时配置

$env:OPENAI_API_KEY = "sk-your-new-api-token"
codex -m gpt-5.5

推荐的 %USERPROFILE%\.codex\config.toml

preferred_auth_method = "apikey"
model_provider = "actorus-newapi"
model = "gpt-5.5"
model_reasoning_effort = "high"

[model_providers.actorus-newapi]
name = "Actorus New API"
base_url = "https://newapi.actorus.top/v1"
requires_openai_auth = true
wire_api = "responses"

配置好后执行下面的命令，把 new-api Token 写入 Codex 本地凭据：

$env:OPENAI_API_KEY = "sk-your-new-api-token"
$env:OPENAI_API_KEY | codex login --with-api-key

测试命令

codex exec -m gpt-5.5 "用一句话回答：连接是否正常？"

如果某些模型不支持 Responses API，可把 wire_api 改为 chat 后再测试。

Claude Code 配置

Claude Code 走 Claude-compatible 接口。Base URL 通常填写域名根地址，不要带 /v1，客户端会请求 /v1/messages。

Windows PowerShell 临时配置

$env:ANTHROPIC_BASE_URL = "https://newapi.actorus.top"
$env:ANTHROPIC_AUTH_TOKEN = "sk-your-new-api-token"
$env:ANTHROPIC_MODEL = "claude-sonnet-4.6"
$env:ANTHROPIC_SMALL_FAST_MODEL = "claude-sonnet-4.6"
claude

如果你的 Claude Code 版本只读取 ANTHROPIC_API_KEY，可以额外设置： 

$env:ANTHROPIC_API_KEY = "sk-your-new-api-token"

测试命令

claude -p --model claude-sonnet-4.6 "用一句话回答：连接是否正常？"

Claude-compatible curl 测试

curl https://newapi.actorus.top/v1/messages ^
  -H "content-type: application/json" ^
  -H "anthropic-version: 2023-06-01" ^
  -H "x-api-key: sk-your-new-api-token" ^
  -d "{\"model\":\"claude-sonnet-4.6\",\"max_tokens\":128,\"messages\":[{\"role\":\"user\",\"content\":\"hello\"}]}"

模型与价格

可用模型以控制台“模型广场”和 /v1/models 返回结果为准。 常用模型示例：

用途 建议模型 备注
Codex 主力 gpt-5.5、gpt-5.4 适合代码生成、重构、复杂任务。
Codex 低成本 gpt-5.4-mini、gpt-5.3-codex 适合日常问答、轻量代码任务。
Claude Code 主力 claude-sonnet-4.6、claude-opus-4.5 适合长上下文代码理解和审阅。
其他模型 deepseek-3.2、qwen3-coder-next、glm-5 实际可用性以账号分组和渠道状态为准。

常见问题

401 Unauthorized

Token 错误、Token 被禁用，或额度不足。请重新复制 new-api 控制台里的 Token。

Codex 连接失败

检查 base_url 是否为 https://newapi.actorus.top/v1， 并确认已通过 codex login --with-api-key 写入 Token。

Claude Code 出现 404 或请求路径异常

检查 ANTHROPIC_BASE_URL 是否为 https://newapi.actorus.top。如果写成 https://newapi.actorus.top/v1，客户端可能会请求到错误路径。

模型不可用

模型可能未分配到你的用户分组，或上游渠道临时不可用。请换一个模型测试，或联系管理员检查渠道。
